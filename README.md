I'm the kind of person who spends three hours automating a five-minute task, then calls it a win. If it can break at 3am, I've probably already broken it on purpose so it doesn't catch me off guard later.

I run a homelab that started as one Raspberry Pi and spiralled into a multi-node cluster with more services than I'll admit to. Most of what I build is infrastructure tooling, deployment patterns, and the kind of glue code that makes distributed systems feel less like herding cats.

**A few things I actually believe:**

- **Infrastructure as code** is great, but infrastructure that *survives* a power outage is better. I care about things that keep running.
- **Self-hosting** isn't a hobby for me — it's how I keep my skills sharp. You learn more from a failed Postgres replication than from ten tutorials.
- **Security** is a process, not a checkbox. I don't trust defaults, I audit dependencies, and I rotate keys like it's a sport.
- **AI is a tool**, not a teammate. I use it to refactor, catch edge cases, and write docs I'd rather not write myself. But I don't ship code I don't understand.

I work across bare metal, VPS, cloud, and edge — whatever fits the job. I've got opinions about package managers, init systems, and why Alpine is underrated.
